# Activities

## Activity 1: Evaluate data requirements.

This part of the activity requires you to evaluate the data requirements based on work brief and input data source.

### Description of the activity

This activity requires you to confirm work brief and tasks to be performed. You are required to analyse the ML requirements and input source data for designing IDS model.

### Steps

1. **Confirm work brief and tasks** to establish objectives with the help of organisational requirements given in the case study section according to organisational policies and procedures.

- Record the work brief documentation having following fields in a document using **Template 1**.
- Objectives
- Technologies to be used
- Applications of each technology in the context of security system

2. **Study and analyse ML requirements** according to cross-industry standard process for data mining (CRISP-DM) methodology, where required:

- Prior knowledge of programming language
- Basic concepts of ML
- Parameters evaluation
- Linear algebra
- **Understanding of phases of CRISP-DM:**
- Business Understanding
- Data Understanding
- Data Preparation
- Modeling
- Evaluation
- Deployment

- Record the ML requirements analysed by you in a document using **Template 2**.

3. **Confirm input machine training data source** using `nsl_kdd_dataset.csv`.

4. **Confirm that data attribute names contain target** according to work brief and review data transformation instructions according to work brief.

- **Data Transformation involves:**
- Data Conversion
- Data Normalization

5. **Confirm that default and non-default training parameters control required learning algorithm** according to work brief. Document the list of default and non-default training parameters using **Template 2**.

## Activity 2: Arrange machine training datasets.

This part of the activity requires you to set training data parameters and select model size according to your work brief.

### Description of the activity

This activity requires you to set training data parameters and select model size according to your work brief.

### Steps

1. **Set machine training data parameters** after splitting training dataset into 70:30 proportions according to work brief and select model size according to work brief.
2. **Use selected parameter over training data** to train the model and apply feature engineering through preprocessing and column filtering (ignore columns judged not relevant for training), including normalisation and other data cleaning techniques as required.
3. **Finalise machine training data procedures** according to work brief and represent the procedure through block diagram in a document using **Template 3**.

## Activity 5: Finalise ML evaluations.

This part of the activity requires you to finalise the ML evaluations using performance parameters.

### Description of the activity

This activity requires you to finalise the ML evaluations using performance parameters.

### Steps

1. **Review target data outputs** according to work brief and adjust or train the model based on any discrepancies of outputs, where required.
2. **Record predictive training accuracy** of ML model according to work brief in a document using **Template 7**.
3. **Run variables through ML model for 100 epochs** and record outputs for each epoch.
4. **Compare outputs returned by ML model** against target data outputs to analyse the gap between training and testing accuracy.
5. **Document metrics and accuracy of ML data predictions** according to organisational policies and procedures using **Template 7**.
